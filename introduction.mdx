Letta Open ADE (Agent Development Environment) is a self-hosted platform built on top of [Letta](https://letta.com) — a framework for building AI agents with persistent memory. ADE adds a complete operator workspace: a REST API for programmatic control, a Next.js frontend with specialized modules, and a suite of automated validation tools to help you develop reliable agents.

## Key capabilities

- **Multiple LLM backends** — connect Doubao (Ark), LM Studio, or any OpenAI-compatible endpoint; switch models without restarting agents
- **Persistent agent memory** — agents remember context across sessions via Letta's core memory blocks (`persona` and `human`), stored in Postgres with pgvector
- **Per-request overrides** — send a single message with a different model or system prompt without changing the agent's stored config
- **Revision history** — every system prompt, persona, or human block edit is logged with timestamps, before/after previews, and character deltas
- **REST API** — full programmatic access to all agent operations via the Agent Platform API on port 8284

## Architecture overview

ADE runs as a set of Docker services:

| Service | Role | Default port |
|---|---|---|
| `letta_server` | Letta agent runtime + REST API | 8283 |
| `letta_db` | Postgres + pgvector for agent state | 5432 |
| `redis` | Message queue for Letta | 6379 |
| `dev_ui` | Agent Platform API (FastAPI) | 8284 |
| `ade_frontend` | ADE web interface (Next.js) | 3000 |

The `dev_ui` service is the primary integration point: it wraps the Letta SDK and exposes a versioned REST API that the ADE frontend and your own scripts can call.
